# Dead Reckoning

A spoiler-safe One Piece atlas.

You tell it where you are in the story — "Chapter 1044" — and it renders the world exactly as you know it, and fogs everything after. Islands you haven't sailed to yet are dark. Crew members who haven't joined yet aren't on the ship.

The existing category leader (onepieceworldmap.com) mapped **where**. Its dataset contains the strings "chapter", "episode" and "saga" zero times. Nobody mapped **when**. The chapter axis is the entire product.

---

## The two-directory data boundary

This is the load-bearing rule of the repo. Violating it is the fastest way to destroy the dataset.

| Directory | Owner | Rule |
|---|---|---|
| `data/raw/` | **machine** | Byte-for-byte mirror of api-onepiece.com. Scripts overwrite freely. Never hand-edit. |
| `data/generated/` | **machine** | Parsed facts from the One Piece Fandom wiki. Scripts overwrite freely. Never hand-edit. |
| `canon/` | **human** | Hand-authored values that exist in **no** upstream source. A script that writes here is a **bug**. |
| `data/canon.json` | **build artifact** | The single merged file the app reads. Generated by `normalize.py`. Committed. |

`canon/` currently holds three files: `crew_joins.json`, `islands.coords.json`, `overrides.json`.

The boundary is enforced **in code**, not just in prose. The sync scripts and the normalizer route every write through a guard that raises on any path under `canon/`:

```
ARCHITECTURE VIOLATION: normalize.py tried to write into canon/
```

That guard is tested and it fires. Do not route around it.

### Three more rules

1. **Zero request-time fetches.** The app never calls an external API. It reads committed JSON. The upstream is a single un-CDN'd Apache box with a bus factor of 1; it is a build-time dependency, never a runtime one.
2. **No database.** The whole corpus is ~4 MB. Committed JSON is the store.
3. **Fail loud.** Unknown status enum, unparseable bounty, a U+FFFD in a string → the normalizer throws. The upstream data is machine-translated French; silent coercion is how "The Chapeau de Paille crew" reaches the UI.

## ⚠ The Jinbe warning — read this twice

**A wiki's "Debut" field is not the chapter a character joined the crew.** It is wrong for 10 out of 10 Straw Hats.

Jinbe **debuts** at episode 430. He **joins** at chapter 976 / episode 977. That is a gap of **547 episodes**.

If crew joins are ever scraped instead of hand-typed, Jinbe stands on the Thousand Sunny for 550 episodes in front of the most pedantic fandom on earth, and the project is dead on arrival.

Therefore:

- **`canon/crew_joins.json` is hand-typed.** It is human-owned. No script produces it.
- **All 10 joins are currently `verified: false`.** They are a best reading, not confirmed against the manga.
- The UI **renders that uncertainty** — it says "10 of 10 join chapters unverified" on screen. It does not hide it.
- `scripts/check_canon.py` has a **`jinbe_test`** tripwire asserting the join/debut gap still exists. If someone "helpfully" repoints crew joins at a scraped Debut field, that test fails. **Do not delete it.**

Every row in `canon.json` carries `source_ref` and `canon_confidence` (`canon` | `derived` | `guess`), NOT NULL — and the confidence is shown in the UI, not just stored. Island positions today are `derived` (393) and `guess` (20); **zero are human-confirmed**, and the app says so out loud.

A related trap, already fixed and worth not re-introducing: the roster is ordered by **join chapter**, not by the order fans recite. Nami is 5th (ch. 95), not 3rd — Usopp (ch. 41) and Sanji (ch. 68) board before she formally joins at the end of Arlong Park.

## Run it

```bash
npm install
npm run dev          # http://localhost:3000
npm run build        # production build
```

Try `/?ch=1`, `/?ch=1044`, `/?ep=1085`. The chapter axis is the whole UI.

## Verify it

```bash
python3 scripts/check_canon.py     # 13 checks, incl. the jinbe_test. Exit 0 = good.
```

## Re-sync the data (rarely; the outputs are committed)

```bash
python3 scripts/sync_api.py        # mirrors api-onepiece.com -> data/raw/
python3 scripts/sync_wiki.py       # facts-only wiki pull -> data/generated/
python3 scripts/normalize.py       # merges raw + generated + canon/ -> data/canon.json
python3 scripts/check_canon.py     # then always re-verify
```

Both syncs are polite by design (sequential, ~1 req/sec, explicit User-Agent, retries, `maxlag=5`) and idempotent — re-runs are cached/skipped unless you pass `--force` / `--refresh`.

**The manifests are the drift detector.** `data/raw/_manifest.json` stores a `sha256` per endpoint. If a sha changes, a record count moved underneath a hand-authored `canon/` row that references it by id. Run the syncs and diff the manifest **before any release**.

### Wiki licensing

Wiki **facts** (numbers, names, ranges) are used; wiki **prose is never copied**. CC-BY-SA 3.0 requires attribution, and the footer carries it. `data/generated/_wiki_cache/` is gitignored on purpose — it contains full rendered article HTML, i.e. the licensed prose. Do not un-ignore it.

## What Phase 1 actually is

The chapter → world join **does not exist upstream**. The API's 50 arcs carry no chapter range; its 1,185 chapters carry no arc. That join is the real work, and it is the moat.

Phase 1 built it:

- **4,088 records** mirrored from 16 API endpoints, zero drift against baseline counts.
- **33 story arcs** with exact chapter *and* episode ranges, in voyage order, chain unbroken.
- **413 locations**, of which **256 are manga-canon with a debut chapter** — the fog-able set.
- Episode ranges are **multi-segment**, because filler interrupts them. Wano is `890-894, 897-906, 908-1028, 1031-1085`. Never membership-test with `[start, end]` — use `episode_segments`. Seven arcs are filler-split.

The fog, measured: ch 1 → 6/256 islands · ch 300 → 80 · ch 600 → 132 · ch 1044 → 236 · ch 1185 → 256.

## Phase 2

- **Verify the crew joins.** Flip `verified: false` → `true` in `canon/crew_joins.json`, one at a time, against the manga. This is the single highest-value hour of human work available in the repo.
- **Confirm island positions.** All 413 are script-placed (`derived`/`guess`). None are human-confirmed. Turning `guess` into `canon` is what makes the map trustworthy.
- **Per-chapter → episode mapping.** Today's map is *arc-granular* and marked `derived`. A true per-chapter map does not exist in either source; it needs a third source. Do not linearly interpolate inside an arc — anime pacing is not linear.
- **Characters on the timeline.** 786 characters carry no chapter linkage at all. Fogging *people* the way islands are fogged is the obvious next axis.
- Launch: see `docs/distribution-reddit.md`. Short version — the site must ship with **zero commercial surface**, and the r/OnePiece post should be framed as an **audit request**, which converts the fandom's pedantry from a threat into free QA.
