# Grand Line Interface

**A spoiler-safe One Piece atlas. Tell it where you are in the story, and it renders the world exactly as you know it — and fogs everything after.**

You give it a chapter ("Chapter 1044") and the map redraws to that moment: islands you've sailed to are charted, the rest are dark; crew who've joined are aboard, the rest are silhouettes. Nothing past your bookmark is ever shown.

The category leader, [The Library of Ohara](https://onepieceworldmap.com), mapped **where** the world is — beautifully, over a decade. Its dataset contains the strings "chapter", "episode" and "saga" *zero times*. Nobody mapped **when**. That's the whole idea here: **Ohara mapped WHERE; this maps WHEN.**

> This is a passion project, built in the open. I'm a founder who uses AI to build and to grow my brand — and part of that brand is sharing what I love. I love One Piece. So this is a gift to the fandom, not a product: free, non-commercial, no ads, no upsell, MIT-licensed, fork it and make it yours.

## Fork it, use it, make it cooler

This repo is meant to be forked. Ideas explicitly on the table for contributors:

- **Set your own bookmark** and share a link to the world as of your chapter.
- **Iceberg lore & theories as map layers** — annotate the chart with theories, poneglyph trails, and "what we think is out there" overlays, chapter-gated so they never spoil.
- **Better marks, better geography, a nicer globe.** The whole visual system is original SVG + generated GeoJSON — no licensed assets to trip over.

If you build something on top of this, open a PR or a discussion. Credit flows both ways.

## Contributing — help place the islands

Island positions are **close, but not hand-verified**. Every island is auto-placed by a belt-model script (`derived`) and rendered as a hollow ring; a human-confirmed position becomes a solid pin (`canon`). I'm not going to hand-route all 400+ myself — the point is to make it **contributable**, so any brave soul who knows One Piece geography can help.

There's a tool for exactly this:

```bash
npm run dev
# then open http://localhost:3000/admin/place   (dev-only; never ships to production)
```

Pick an island (the list is sorted worst-confidence-first), click the chart where it belongs, and the position is written straight into `canon/islands.coords.json` — **a clean ~4-line diff you can open as a PR**. Keys: `n` next island, `1/2/3` confidence.

**Even better than clicking:** if you know of an authoritative, license-clean **dataset** of One Piece island coordinates (Vivre Card data, a community-maintained map with a compatible license, etc.), open an issue — the goal is to drive accuracy from data so **nobody has to human-review anything**. The `/admin/place` tool is the fallback, not the aspiration.

Same invitation applies to **crew flags, character placements, and lore layers** — see the roadmap.

## How it's built (and the rules that keep it honest)

### The two-directory data boundary

The load-bearing rule of the repo. Violating it is the fastest way to destroy the dataset.

| Directory | Owner | Rule |
|---|---|---|
| `data/raw/` | **machine** | Byte-for-byte mirror of api-onepiece.com. Scripts overwrite freely. Never hand-edit. |
| `data/generated/` | **machine** | Parsed facts from the One Piece Fandom wiki. Scripts overwrite freely. Never hand-edit. |
| `canon/` | **human** | Hand-authored values that exist in **no** upstream source. A script that writes here is a **bug**. |
| `data/canon.json` | **build artifact** | The single merged file the app reads. Generated by `normalize.py`. Committed. |

The boundary is enforced **in code**: the sync scripts and normalizer route every write through a guard that raises on any path under `canon/`. That guard is tested and it fires. Don't route around it.

Three more rules:

1. **Zero request-time fetches.** The app never calls an external API — it reads committed JSON. The upstream is a single un-CDN'd box with a bus factor of 1; it is a build-time dependency, never a runtime one.
2. **No database.** The whole corpus is ~4 MB. Committed JSON is the store.
3. **Fail loud.** Unknown status enum, unparseable bounty, a U+FFFD in a string → the normalizer throws. The upstream data is machine-translated from French; silent coercion is how "The Chapeau de Paille crew" reaches the UI.

### ⚠ The Jinbe warning — read this twice

**A wiki's "Debut" field is not the chapter a character joined the crew.** It is wrong for 10 out of 10 Straw Hats. Jinbe **debuts** at episode 430 and **joins** at chapter 976 / episode 977 — a gap of **547 episodes**. Scrape Debut into a join column and Jinbe stands on the Thousand Sunny for 550 episodes in front of the most pedantic fandom on earth.

Therefore `canon/crew_joins.json` is **hand-typed and human-owned**, every join is currently **`verified: false`** (a best reading, not yet confirmed against the manga), the UI **renders that uncertainty** out loud, and `scripts/check_canon.py` has a **`jinbe_test`** tripwire. Do not delete it.

Every authored row carries `source_ref` and `canon_confidence` (`canon` | `derived` | `guess`), NOT NULL, and the confidence is shown in the UI, not just stored.

### Re-syncing the data (rarely — outputs are committed)

```bash
python3 scripts/sync_api.py        # mirrors api-onepiece.com  -> data/raw/
python3 scripts/sync_wiki.py       # facts-only wiki pull      -> data/generated/
python3 scripts/normalize.py       # raw + generated + canon/  -> data/canon.json
python3 scripts/check_canon.py     # then always re-verify
```

Both syncs are polite (sequential, ~1 req/sec, explicit User-Agent, retries) and idempotent. `data/raw/_manifest.json` stores a `sha256` per endpoint — the drift detector. `data/generated/_wiki_cache/` is gitignored on purpose: it holds full rendered article HTML (licensed prose). Only facts live in the repo.

## Roadmap

**Shipped**
- **The Straw Hat journey** — a route line that draws itself as the chapter climbs (ch 1 → the current arc), and the ship morphing barrel → small boat → Going Merry → Thousand Sunny at the right chapters.
- **Real art, chapter-gated** *(Phase 6)* — real Jolly Rogers, character portraits, ship renders, Devil Fruits and island images, downscaled and served from `public/art/`. Official artwork © the rights holders, used as attributed fan reference and **excluded from the MIT license** (see [`public/art/README.md`](public/art/README.md)); every file carries a source-URL receipt in `data/generated/art_manifest.json`. The original SVG marks below stay in the code as fallbacks — delete `public/art/` and the atlas still works.
- **Original SVG marks** — hand-drawn Jolly Rogers and ship glyphs from primitives, no traced art; the fallback layer whenever no real art exists (the barrel, the nameless first boat) or a takedown ever lands.
- **A cartographic chart** — the Grand Line as a sea-lane between menacing Calm Belts, the Red Line as a continent, ocean depth + vellum grain, a compass rose, on both globe and flat.
- **`/admin/place`** — the dev-only coordinate tool (see Contributing).
- **Crews & characters on the map, chapter-gated** — 10 crews (the Yonko, Kid, Law/Heart, Kuja, Donquixote, Buggy) fly their Jolly Rogers at authored *presence windows*, so they **move**: Whitebeard sails the New World, appears at Marineford for the war, and is gone after it. 8 Warlords and ~14 named crew members ride as portrait rings over ink orbs, a "who sails here" toggle, and live crew chips in the legend. All windows hand-authored in `canon/crew_presence.json`, `verified:false` until manga-checked — same honesty as every pin.

- **The By Crew / By Devil Fruit / By Haki lens** *(Phase 6A)* — recolor the world by what a character is: crew affiliation, revealed fruit nature, or highest revealed haki. `?lens=` in the URL; every lens shows revealed facts only, so at ch. 1 each shows exactly nothing.
- **Every dot a door** *(Phase 8)* — wanted-poster entry pages for `/character/[slug]`, `/crew/[slug]`, `/fruit/[slug]`, with OG cards.
- **The third dimension** *(Phase 9)* — a three.js GLB layer on the MapLibre globe; 11 runtime island models (flag-gated, off in prod).

**Next**
- **The canon events layer** — the duels, wars, deaths and declarations as chapter-gated map facts (the Zoro–Mihawk duel at the Baratie, the war at Marineford), hand-authored in `canon/events.json` with the same honesty rules as every pin.
- **Expand the roster** — the rest of the Worst Generation (Hawkins, Drake, Bonney, Capone, Urouge, Apoo), Sun Pirates, Baroque Works. The schema is done; it's authoring + one Jolly Roger each. `scripts/seed_presence.py` prints a scaffold.
- **Verify the canon** — flip `verified: false → true` on crew joins, ships, presence windows, and island positions. The full debt is enumerated by `scripts/verification_queue.py` → [`data/review/verification-queue.md`](data/review/verification-queue.md).
- **Iceberg lore & theory layers** — see "Fork it" above.

The cross-track master plan (three parallel builders, one reconciliation point) lives in [`docs/ROADMAP.md`](docs/ROADMAP.md).
